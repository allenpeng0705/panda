Roadmap

Panda's development roadmap and future plans

Development Phases

Panda is being developed in several phases, each focusing on specific capabilities and features.

Phase 1

Heart

Rust proxy core with HTTP/1.1 + HTTP/2-ready streaming (SSE), backpressure, upstream failover hooks, and configuration.

  • • Basic proxy functionality
  • • OpenAI-compatible API
  • • Streaming support
  • • Configuration system
Completed
Phase 2

Skin

Wasm plugin system with load, isolate, host ABI, and hot reload capabilities.

  • • Wasmtime integration
  • • Plugin ABI
  • • Hot reload
  • • Plugin examples
Completed
Phase 3

Bouncer

JWT / OIDC, scoped agent tokens, PII scrubbing path, TPM, and injection heuristics.

  • • JWT authentication
  • • PII detection and redaction
  • • Token-based budgeting (TPM)
  • • Prompt safety
Completed
Phase 4

Brain

Intent routing (ONNX), semantic cache, universal adapters, MCP host, proof-of-intent on tools, and optional context enrichment.

  • • Intent-based routing
  • • Semantic caching
  • • MCP orchestration
  • • Universal adapters
Completed
Phase 5

Enterprise Finish

OpenTelemetry, packaging, Kubernetes, health probes, load + soak tests, and allocator tuning.

  • • OpenTelemetry integration
  • • Kubernetes deployment
  • • Performance tuning
  • • Production hardening
Completed

Frequently Asked Questions

Panda is currently in active development. We're working towards a 1.0 release once all core features are implemented and thoroughly tested. Stay tuned for updates on the GitHub repository.

You can request features by opening an issue on the GitHub repository. Please include a clear description of the feature, why it's needed, and any relevant use cases.

Panda is specifically designed for AI traffic, with features like token-based budgeting, semantic caching, MCP tool orchestration, and prompt safety that are not typically found in generic API gateways.

Panda is being actively developed and improved. While it's suitable for testing and development environments, we recommend caution when using it in production until it reaches a stable 1.0 release.

You can stay updated by following the GitHub repository, joining the Discord community, and checking the release notes for new versions.
