QuickStart Guide

Get up and running with Panda in minutes

Prerequisites

Before you begin, make sure you have the following:

  • Rust toolchain (stable) or Docker
  • A model server URL (e.g., Ollama on http://127.0.0.1:11434 or a cloud provider's OpenAI-compatible base URL)

1. Clone the Repository

Start by cloning the Panda repository:

$ git clone https://github.com/allenpeng0705/panda.git
$ cd panda

2. Configure Panda

Copy the example configuration file and edit it to your needs:

$ cp panda.example.yaml panda.yaml

Edit panda.yaml to set your upstream provider:

# Example: Local Ollama
upstream: http://127.0.0.1:11434

# Example: OpenAI
# upstream: https://api.openai.com

The default listen address is 127.0.0.1:8080.

3. Run Panda

Using Cargo (Rust):

$ cargo run -p panda-server -- panda.yaml

For more verbose logs:

$ RUST_LOG=info cargo run -p panda-server -- panda.yaml

Using Docker:

$ docker build -t panda:latest .
$ docker run --rm -p 8080:8080 \
  -v "$(pwd)/panda.yaml:/app/panda.yaml:ro" \
  panda:latest /app/panda.yaml

4. Verify the Installation

Check if Panda is running:

$ curl -s http://127.0.0.1:8080/health

Send a test chat request:

$ curl -s http://127.0.0.1:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model": "gpt-4o-mini", "messages": [{"role": "user", "content": "hello from Panda"}]}'

Adjust the model parameter to match your upstream provider.

5. Enable the Developer Console (Optional)

For a live debug UI:

$ PANDA_DEV_CONSOLE_ENABLED=true cargo run -p panda-server -- panda.yaml

Open http://127.0.0.1:8080/console in your browser.

Troubleshooting

Common Issues

  • Connection Refused

    Make sure your upstream server is running and accessible.

  • Authentication Errors

    Check your API keys and authentication configuration.

  • Model Not Found

    Ensure the model name you're using exists in your upstream provider.

Useful Commands

# Check logs
RUST_LOG=debug cargo run -p panda-server -- panda.yaml

# Test with a different upstream
PANDA_LISTEN_OVERRIDE=0.0.0.0:8080 cargo run -p panda-server -- panda.yaml

# Enable OTLP tracing
OTEL_EXPORTER_OTLP_ENDPOINT=http://127.0.0.1:4318/v1/traces cargo run -p panda-server -- panda.yaml
