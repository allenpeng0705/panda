Observability

Monitor, trace, and log your AI traffic

Observability Features

Panda provides comprehensive observability tools to help you monitor and debug your AI traffic.

Metrics

Prometheus-compatible metrics for monitoring Panda's performance and usage.

  • • Request counts and latencies
  • • Token usage (TPM)
  • • Cache hit/miss rates
  • • Plugin execution times
  • • Error rates and types

Tracing

OpenTelemetry tracing for end-to-end request visibility.

  • • Distributed tracing
  • • Request lifecycle spans
  • • Upstream request tracking
  • • MCP tool execution tracing
  • • Plugin execution tracing

Logging

Structured JSON logging for detailed operational insights.

  • • Request/response details
  • • Error messages and stack traces
  • • Plugin execution events
  • • Token usage information
  • • Configuration changes

Metrics

Metrics Endpoint

Panda exposes metrics in Prometheus text format at the /metrics endpoint.

# Example: Accessing metrics
curl http://localhost:8080/metrics

# With authentication (if enabled)
curl -H "x-panda-admin-secret: $PANDA_OPS_SECRET" http://localhost:8080/metrics

Key Metrics

  • • panda_http_requests_total - Total HTTP requests
  • • panda_http_request_duration_seconds - Request duration histogram
  • • panda_tpm_tokens_total - Total tokens processed
  • • panda_semantic_cache_hits_total - Cache hits
  • • panda_semantic_cache_misses_total - Cache misses
  • • panda_plugin_execution_duration_seconds - Plugin execution time
  • • panda_mcp_tool_calls_total - MCP tool calls

Prometheus Configuration

# prometheus.yml
scrape_configs:
  - job_name: 'panda'
    scrape_interval: 15s
    static_configs:
      - targets: ['panda:8080']
    # If authentication is enabled
    basic_auth:
      username: ''
      password: 'your-secret'
    # Or custom headers
    headers:
      x-panda-admin-secret: 'your-secret'

Tracing

Panda supports OpenTelemetry tracing for distributed request tracking.

Configuration

Set the following environment variables to enable tracing:

export OTEL_EXPORTER_OTLP_ENDPOINT=http://localhost:4318/v1/traces
export PANDA_OTEL_SERVICE_NAME=panda-gateway
export PANDA_OTEL_TRACE_SAMPLING_RATIO=0.2

Supported Backends

  • • Jaeger
  • • Grafana Tempo
  • • Zipkin
  • • Cloud APM services (AWS X-Ray, Google Cloud Trace, etc.)

Trace Example

A typical trace will include spans for:

  • • Request reception
  • • Authentication and authorization
  • • Plugin execution
  • • Semantic cache lookup
  • • Upstream request
  • • MCP tool calls (if applicable)
  • • Response preparation

Logging

Log Configuration

Set the log level using the RUST_LOG environment variable:

# Basic logging
RUST_LOG=info cargo run -p panda-server -- panda.yaml

# Debug logging
RUST_LOG=debug cargo run -p panda-server -- panda.yaml

# Trace logging (very verbose)
RUST_LOG=trace cargo run -p panda-server -- panda.yaml

Log Format

Panda uses structured JSON logging by default:

{
  "timestamp": "2023-10-01T12:00:00Z",
  "level": "INFO",
  "message": "Request completed",
  "fields": {
    "path": "/v1/chat/completions",
    "method": "POST",
    "status": 200,
    "duration_ms": 1234,
    "tokens": 456,
    "model": "gpt-4o-mini",
    "cache_hit": false
  }
}

Developer Console

The Developer Console provides a live debug UI for monitoring request flow and MCP activity.

Enabling the Console

# Enable the developer console
PANDA_DEV_CONSOLE_ENABLED=true cargo run -p panda-server -- panda.yaml

# Access the console at:
# http://localhost:8080/console

Console Features

  • • Real-time request monitoring
  • • MCP tool activity visualization
  • • Token usage tracking
  • • Plugin execution details
  • • Cache hit/miss information

Health Endpoints

/health

Liveness probe - indicates if the process is running.

$ curl http://localhost:8080/health
{"status": "ok"}

/ready

Readiness probe - indicates if Panda is ready to accept requests.

$ curl http://localhost:8080/ready
{"status": "ready"}

/tpm/status

Token budget status and usage information.

$ curl http://localhost:8080/tpm/status
{"budgets": {...}}

/mcp/status

MCP tool server status and activity.

$ curl http://localhost:8080/mcp/status
{"servers": {...}}

/metrics

Prometheus metrics endpoint.

$ curl http://localhost:8080/metrics
# HELP panda_http_requests_total Total HTTP requests
# TYPE panda_http_requests_total counter
panda_http_requests_total{method="POST",path="/v1/chat/completions"} 123

Datadog Integration

Configure Datadog to collect Panda metrics:

# datadog.yaml
prometheus_check:
  instances:
    - prometheus_url: http://panda:8080/metrics
      headers:
        x-panda-admin-secret: your-secret
      namespace: panda
      metrics:
        - panda_http_requests_total
        - panda_tpm_tokens_total
        - panda_semantic_cache_hits_total
