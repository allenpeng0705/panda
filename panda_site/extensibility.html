Extensibility

Extend Panda with Wasm plugins

Wasm Plugin System

Panda uses WebAssembly (Wasm) as its primary extension mechanism, allowing you to add custom policy hooks, transforms, and integrations without forking the core.

Benefits of Wasm Plugins

  • • Sandboxed execution: Plugins run in an isolated environment with resource limits
  • • Language flexibility: Write plugins in Rust, Go (via TinyGo), or any language that compiles to Wasm
  • • Hot reload: Update plugins without restarting Panda
  • • Performance: Near-native execution speed
  • • Security: Strict access controls to host functions

Plugin Lifecycle Hooks

  • • panda_on_request: Process request headers
  • • panda_on_request_body: Process request body
  • • panda_on_response_chunk: Process response chunks

Plugins can modify headers, replace request bodies, and enforce custom policies.

Creating Rust Plugins

Setup

# Create a new Rust library project
cargo new --lib my-panda-plugin
cd my-panda-plugin

# Edit Cargo.toml
cat > Cargo.toml << 'EOF'
[package]
name = "my-panda-plugin"
version = "0.1.0"
edition = "2021"

[lib]
crate-type = ["cdylib"]

dependencies = {
    "panda-pdk" = { path = "../crates/panda-pdk" }
}
EOF

Plugin Implementation

// src/lib.rs
use panda_pdk::*;

#[no_mangle]
pub extern "C" fn panda_abi_version() -> i32 {
    0 // Must match host ABI version
}

#[no_mangle]
pub extern "C" fn panda_on_request() -> i32 {
    // Example: Add a custom header
    let name = "X-Custom-Header";
    let value = "my-value";
    panda_set_header(name.as_ptr(), name.len() as i32, value.as_ptr(), value.len() as i32);
    0 // Allow the request
}

#[no_mangle]
pub extern "C" fn panda_on_request_body(ptr: i32, len: i32) -> i32 {
    // Example: Read and modify request body
    let body = unsafe { std::slice::from_raw_parts(ptr as *const u8, len as usize) };
    let body_str = std::str::from_utf8(body).unwrap_or("");
    
    // Modify body as needed
    let modified_body = body_str.replace("secret", "[REDACTED]");
    
    // Set the modified body
    panda_set_body(modified_body.as_ptr(), modified_body.len() as i32);
    0 // Allow the request
}

Building the Plugin

# Build for Wasm target
rustup target add wasm32-wasi
cargo build --target wasm32-wasi --release

# The plugin will be at:
target/wasm32-wasi/release/my_panda_plugin.wasm

Creating Go Plugins

You can also write Panda plugins in Go using TinyGo, which compiles to WebAssembly.

Setup

# Install TinyGo
# Follow instructions at https://tinygo.org/getting-started/

# Create a new Go project
mkdir my-go-plugin
cd my-go-plugin
go mod init my-go-plugin

# Add panda-pdk
go get github.com/allenpeng0705/panda/crates/panda-pdk/go/panda

Plugin Implementation

// main.go
package main

import (
    "github.com/allenpeng0705/panda/crates/panda-pdk/go/panda"
)

//export panda_abi_version
func panda_abi_version() int32 {
    return 0 // Must match host ABI version
}

//export panda_on_request
func panda_on_request() int32 {
    // Example: Add a custom header
    panda.SetHeader("X-Go-Plugin", "Hello from Go")
    return 0 // Allow the request
}

//export panda_on_request_body
func panda_on_request_body(ptr int32, len int32) int32 {
    // Example: Read request body
    body := panda.GetBody(ptr, len)
    // Modify body as needed
    modified := []byte("Modified by Go plugin")
    panda.SetBody(modified)
    return 0 // Allow the request
}

func main() {
    // Required for TinyGo
}

Building the Plugin

# Build with TinyGo
tinygo build -o my-go-plugin.wasm -target wasi main.go

Configuring Plugins

Plugin Configuration in panda.yaml

plugins:
  # Path to directory containing Wasm plugins
  directory: "./plugins"
  # Execution timeout per plugin (milliseconds)
  execution_timeout_ms: 100
  # Maximum request body size for body hooks (bytes)
  max_request_body_bytes: 1048576
  # Enable hot reload of plugins
  hot_reload: true
  # Hot reload interval (milliseconds)
  reload_interval_ms: 5000
  # Fail closed if plugin execution fails
  fail_closed: true
  # Wasm instance pool size
  instance_pool_size: 4

Plugin Return Codes

Code Meaning Host Behavior
0 Allow Continue processing
1 Reject (policy denied) Return 403 Forbidden
2 Reject (malformed request) Return 400 Bad Request
Other non-zero Plugin-specific error Return 502 Bad Gateway

Host Imports

  • • panda_set_header: Add or modify request headers
  • • panda_set_body: Replace the request body

These functions are available to plugins for modifying requests before they are sent to the upstream.

Plugin Development Best Practices

Security

  • • Validate all input from the request
  • • Avoid memory leaks
  • • Be mindful of resource usage
  • • Don't perform blocking operations
  • • Limit external dependencies

Performance

  • • Keep plugin logic simple and fast
  • • Avoid unnecessary allocations
  • • Use efficient algorithms
  • • Test with realistic payloads
  • • Monitor execution time
