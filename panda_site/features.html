Features

Comprehensive capabilities for AI traffic management

Core Features

OpenAI-Compatible API

Panda presents an OpenAI-compatible HTTP API to clients, making it easy to integrate with existing applications and SDKs.

  • • Chat completions endpoint (/v1/chat/completions)
  • • Streaming support via Server-Sent Events (SSE)
  • • Compatible with OpenAI SDKs and clients
  • • Adapter path for other provider protocols

Token-Based Budgeting (TPM)

Control costs with token-per-minute (TPM) budgeting and limits.

  • • Per-subject token budgets
  • • Optional Redis integration for shared counters
  • • /tpm/status endpoint for budget visibility
  • • Configurable enforcement policies

Semantic Cache

Reduce costs and latency with vector similarity-based caching.

  • • Vector similarity over prompts
  • • Memory or Redis backend options
  • • Configurable similarity threshold
  • • Cache hit/miss metrics

MCP Orchestration

Model Context Protocol for tool orchestration and agent capabilities.

  • • Tool call interception and execution
  • • Multi-round follow-ups
  • • Streaming tool responses
  • • Unified tool surface for LLMs

Security & Safety

Comprehensive security features to protect your AI systems.

  • • JWT/JWKS authentication
  • • Prompt safety and injection protection
  • • PII detection and redaction
  • • Wasm plugins for custom security policies

Observability

Detailed metrics, logs, and traces for operational visibility.

  • • Prometheus metrics endpoint
  • • OpenTelemetry tracing
  • • Structured JSON logs
  • • Health and readiness endpoints

Enterprise Features

Optional enterprise-grade capabilities for larger organizations.

SSO Integration

Single Sign-On for the developer console using enterprise identity providers.

  • • Okta integration
  • • Microsoft Entra integration
  • • OIDC compliance
  • • Role-based access control

Hierarchical Budgets

Multi-level budget hierarchies for teams and departments.

  • • Department-level budgets
  • • Team-level allocations
  • • User-level limits
  • • Budget rollup reporting

Model Failover

Automatic failover between model providers for reliability.

  • • Ordered failover chain
  • • Health checks
  • • Error-based failover
  • • Load balancing across providers

Performance Features

Stream Optimization

Panda is built for efficient streaming of LLM responses:

  • • SSE (Server-Sent Events) support
  • • Minimal copying of stream data
  • • Stream timeouts and guards
  • • Backpressure handling

Scalability

Designed for horizontal scaling:

  • • Stateless core design
  • • Optional shared state via Redis
  • • Kubernetes-ready
  • • Connection-based scaling

Runtime Performance

  • • Rust-based implementation
  • • Async I/O with Tokio
  • • Optional mimalloc for long-lived workloads
  • • rustls for TLS termination

Resource Efficiency

  • • Low memory footprint
  • • CPU-efficient streaming
  • • Wasm instance pooling
  • • Configurable resource limits

Developer Experience

Developer Console

Live debug UI for request flow and MCP activity.

  • • Real-time request monitoring
  • • MCP tool activity
  • • Token usage visualization
  • • Configurable access control

Configuration

GitOps-friendly YAML configuration.

  • • Single YAML config file
  • • Environment variable overrides
  • • Hot reload for plugins
  • • Clear schema documentation

API Documentation

Comprehensive API documentation and examples.

  • • OpenAPI specification
  • • Example requests
  • • SDK integration guides
  • • Troubleshooting documentation

Feature Comparison

How Panda compares to generic API gateways

Feature Panda Generic API Gateway
AI-specific protocols ✓ OpenAI-compatible, SSE streaming ✗ Limited support
Token-based budgeting ✓ TPM with Redis integration ✗ Usually RPS-only
Semantic cache ✓ Vector similarity-based ✗ Not available
MCP tool orchestration ✓ Built-in ✗ Not available
Prompt safety ✓ Built-in protection ✗ Not available
PII scrubbing ✓ Built-in ✗ Limited or plugin-only
Wasm plugins ✓ Rust and Go support ✗ Often Lua-only
Intent-aware routing ✓ Based on prompt content ✗ Path-based only
Stream optimization ✓ Purpose-built for SSE ✗ Generic HTTP handling
Stateless design ✓ No mandatory database ✗ Often requires control plane DB
