Security

Protect your AI systems with comprehensive security features

Security Features

Panda provides a comprehensive set of security features to protect your AI systems and data.

Authentication

Secure authentication mechanisms to control access to your AI services.

  • • JWT/JWKS validation
  • • Trusted gateway headers
  • • API key support
  • • Admin endpoint protection

Authorization

Fine-grained access control to ensure users and applications only access what they're allowed to.

  • • JWT scope validation
  • • Per-route access control
  • • Tool call authorization
  • • Rate limiting

Data Protection

Protect sensitive data in transit and at rest.

  • • TLS termination
  • • PII detection and redaction
  • • Encrypted configuration
  • • Secure secret management

Authentication

JWT Authentication

Configure JWT authentication in panda.yaml:

identity:
  # Require JWT for all requests
  require_jwt: true
  # HS256 secret (from environment)
  jwt_hs256_secret_env: "PANDA_JWT_SECRET"
  # JWKS URL for RS256 tokens
  jwt_jwks_url: "https://auth.example.com/.well-known/jwks.json"
  # Expected audience
  jwt_audience: "panda-api"
  # Expected issuer
  jwt_issuer: "https://auth.example.com"
  # Claims mapping
  jwt_claims:
    subject: "sub"
    scopes: "scp"
    email: "email"

Trusted Gateway

When running behind KONG or another API gateway:

trusted_gateway:
  # Attestation header name
  attestation_header: "x-panda-gateway-attestation"
  # Identity header name
  identity_header: "x-user-id"
  # Scopes header name
  scopes_header: "x-user-scopes"
  # Secret for attestation (from environment)
  secret_env: "PANDA_TRUSTED_GATEWAY_SECRET"

# Environment variable
export PANDA_TRUSTED_GATEWAY_SECRET="your-secret-key"

Authorization

Scope-Based Authorization

Panda can validate JWT scopes to control access to specific routes:

routes:
  - path: "/v1/chat/completions"
    methods: ["POST"]
    required_scopes: ["chat:completions"]
  - path: "/v1/embeddings"
    methods: ["POST"]
    required_scopes: ["embeddings:create"]
  - path: "/admin/*"
    methods: ["GET", "POST", "PUT", "DELETE"]
    required_scopes: ["admin:access"]

Rate Limiting

Control the rate of requests to prevent abuse:

routes:
  - path: "/v1/chat/completions"
    methods: ["POST"]
    # Requests per second per IP
    rps_limit: 10
    # Requests per minute per subject
    rpm_limit: 100
    # Token budget per minute
    tpm_limit: 10000

Data Protection

PII Detection and Redaction

Panda can detect and redact personally identifiable information:

shield:
  # Enable PII detection
  pii_detection: true
  # PII types to detect
  pii_types:
    - email
    - phone
    - credit_card
    - social_security
    - address
  # Redaction strategy
  pii_redaction:
    mode: "mask"
    mask_character: "*"
    mask_length: 4

TLS Configuration

Configure TLS for secure connections:

tls:
  # Path to certificate PEM file
  cert_pem: "/path/to/cert.pem"
  # Path to private key PEM file
  key_pem: "/path/to/key.pem"
  # TLS versions to enable
  min_version: "TLSv1.2"
  # Cipher suites
  cipher_suites: ["TLS_AES_256_GCM_SHA384", "TLS_CHACHA20_POLY1305_SHA256"]

Prompt Safety

Panda includes built-in protection against prompt injection attacks and other safety issues.

Prompt Injection Protection

shield:
  # Enable prompt safety
  prompt_safety: true
  # Block high-risk prompts
  block_high_risk: true
  # Score threshold for blocking
  risk_threshold: 0.7
  # Custom blocked patterns
  blocked_patterns:
    - "Ignore previous instructions"
    - "System prompt"
    - "You are a malicious AI"

Wasm Plugins for Custom Safety

Use Wasm plugins to implement custom safety checks:

// Example: Custom prompt safety plugin
#[no_mangle]
pub extern "C" fn panda_on_request_body(ptr: i32, len: i32) -> i32 {
    let body = unsafe { std::slice::from_raw_parts(ptr as *const u8, len as usize) };
    let body_str = std::str::from_utf8(body).unwrap_or("");
    
    // Check for unsafe patterns
    if body_str.contains("Ignore previous instructions") {
        return 1; // Reject
    }
    
    // Check for sensitive information
    if body_str.contains("password") || body_str.contains("secret") {
        return 1; // Reject
    }
    
    0 // Allow
}

Wasm Plugin Security

Sandboxing

Wasm plugins run in a sandboxed environment with strict limits:

plugins:
  # Execution timeout per plugin (milliseconds)
  execution_timeout_ms: 100
  # Maximum memory per plugin (megabytes)
  max_memory_mb: 16
  # Maximum request body size for body hooks (bytes)
  max_request_body_bytes: 1048576
  # Fail closed if plugin execution fails
  fail_closed: true
  # Wasm instance pool size
  instance_pool_size: 4

Host Function Access

Plugins have limited access to host functions:

  • • panda_set_header - Add or modify request headers
  • • panda_set_body - Replace the request body

Plugins cannot access the file system, network, or other system resources.

Security Best Practices

Deployment Security

  • • Run Panda as a non-root user
  • • Set resource limits in containers
  • • Use a read-only filesystem where possible
  • • Restrict network access with firewalls
  • • Keep Panda and dependencies updated

Configuration Security

  • • Use environment variables for secrets
  • • Never commit secrets to git
  • • Use Kubernetes Secrets or a secret manager
  • • Rotate secrets regularly
  • • Validate configuration before deployment

Operational Security

  • • Enable audit logging
  • • Monitor for unusual activity
  • • Set up alerts for security events
  • • Perform regular security reviews
  • • Test security controls regularly

Threat Model

Potential Threats

  • • Prompt injection: Malicious prompts that attempt to override system instructions
  • • Data exfiltration: Attempts to extract sensitive information
  • • Denial of service: Excessive requests to consume resources
  • • Tool abuse: Unauthorized use of MCP tools
  • • Authentication bypass: Attempts to access without valid credentials
  • • Configuration vulnerabilities: Insecurely configured settings

Mitigations

  • • Prompt safety: Built-in protection against injection attacks
  • • PII redaction: Automatic detection and redaction of sensitive data
  • • Rate limiting: Control the rate of requests
  • • Tool authorization: Validate tool calls against user intent
  • • Strong authentication: JWT/JWKS and trusted gateway headers
  • • Secure configuration: Best practices for secret management
