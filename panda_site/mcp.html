Model Context Protocol (MCP)

Tool orchestration for AI agents

What is MCP?

The Model Context Protocol (MCP) is Panda's mechanism for orchestrating tool calls between AI models and external services. It allows LLMs to use tools to perform actions, access data, and interact with external systems. MCP is integrated directly into the Panda gateway, providing a unified tool surface for AI agents.

Key Benefits

  • • Unified tool surface: Expose multiple tools to LLMs through a single interface
  • • Tool discovery: Automatic discovery and aggregation of tools from multiple servers
  • • Tool call execution: Dispatch tool calls to the correct servers with timeout and payload bounds
  • • OpenAI mapping: Automatically map MCP tools to OpenAI function calling format
  • • Round limits: Enforce global and per-request round limits for tool use
  • • Streaming support: Support for streaming tool responses
  • • Security: Validate tool calls against user intent and permissions

How It Works

  1. 1. Client sends a request to Panda
  2. 2. Panda aggregates tools from all configured MCP servers
  3. 3. Panda forwards the request with available tools to the LLM
  4. 4. LLM responds with tool calls
  5. 5. Panda intercepts and executes the tool calls
  6. 6. Panda sends the tool results back to the LLM
  7. 7. LLM generates a final response based on the tool results
  8. 8. Panda returns the response to the client

MCP Runtime

The MCP runtime in Panda manages connections to tool servers and handles tool call orchestration.

  • • Holds HashMap of Arc<dyn McpClient>
  • • Implements list_all_tools and call_tool methods
  • • Manages tool call timeouts and payload bounds
  • • Enforces round limits for tool use

MCP Clients

Different client implementations for various tool server types.

  • • StdioMcpClient: JSON-RPC over stdio
  • • McpHttpRemoteClient: Remote MCP over HTTP
  • • HttpToolClient: Declarative REST tools via egress

OpenAI Mapping

Maps MCP tools to OpenAI function calling format.

  • • Converts MCP tools to OpenAI tools[]
  • • Uses mcp_{server}_{tool} naming convention
  • • Handles function name sanitization
  • • Supports structured tool results

Configuring MCP

MCP Configuration in panda.yaml

mcp:
  # Enable MCP
  enabled: true
  # Tool servers
  servers:
    - name: "stdio_server"
      enabled: true
      command: ["./path/to/server.py"]
      args: ["--option", "value"]
    - name: "http_tool_server"
      enabled: true
      http_tool:
        egress_profile: internal_rest
        base_path: /v1/api
  # Advanced settings
  fail_open: true
  tool_timeout_ms: 30000
  max_tool_payload_bytes: 1048576
  max_tool_rounds: 10
  advertise_tools: true
  # Stream probe settings
  stream_probe_bytes: 16384
  probe_window_seconds: 60

Tool Execution Types

  • Stdio MCP: Tool server communicates via standard input/output 
    # Example stdio tool server
#!/usr/bin/env python3
import json
import sys

while True:
    # Read tool call from stdin
    line = sys.stdin.readline()
    if not line:
        break
    
    # Parse tool call
    tool_call = json.loads(line)
    
    # Process tool call
    result = process_tool_call(tool_call)
    
    # Write result to stdout
    print(json.dumps(result))
    sys.stdout.flush()
  • HTTP Tool (REST): Declarative REST tools via API gateway egress
  • Remote MCP: Remote MCP server via HTTP JSON-RPC

MCP in Action

Example: Using Postgres Tool

# 1. Configure Postgres tool server in panda.yaml
mcp:
  enabled: true
  servers:
    - name: "postgres"
      type: "stdio"
      command: ["./panda-mcp-registry/postgres/server.py"]
      env:
        - name: "DATABASE_URL"
          value: "postgres://user:pass@localhost:5432/db"

# 2. Client sends request with tool use
def send_request():
    response = openai.ChatCompletion.create(
        model="gpt-4o-mini",
        messages=[
            {"role": "system", "content": "You are a helpful assistant with access to a Postgres database. Use the postgres tool to query data."},
            {"role": "user", "content": "How many users are in the database?"}
        ],
        tools=[
            {
                "type": "function",
                "function": {
                    "name": "postgres_query",
                    "description": "Execute a SQL query against the database",
                    "parameters": {
                        "type": "object",
                        "properties": {
                            "query": {
                                "type": "string",
                                "description": "SQL query to execute"
                            }
                        },
                        "required": ["query"]
                    }
                }
            }
        ]
    )
    return response

# 3. Panda intercepts tool call and executes it
# 4. LLM receives tool result and generates response

Tool Call Flow

  1. 1. LLM generates a tool call with function name and parameters
  2. 2. Panda routes the tool call to the appropriate server
  3. 3. Tool server executes the function and returns a result
  4. 4. Panda sends the result back to the LLM
  5. 5. LLM generates a final response based on the result

Security Considerations

  • • Validate tool calls against user intent
  • • Set appropriate timeouts for tool execution
  • • Limit tool server permissions
  • • Monitor tool usage for异常 patterns
  • • Use environment variables for secrets

MCP Best Practices

Tool Design

  • • Keep tool functions focused and specific
  • • Provide clear descriptions for LLMs
  • • Define well-structured parameters
  • • Return structured, machine-readable results

Performance

  • • Keep tool execution fast
  • • Set appropriate timeouts
  • • Use streaming for long-running tools
  • • Cache frequently used results

Reliability

  • • Handle errors gracefully
  • • Implement retries for transient failures
  • • Monitor tool server health
  • • Provide clear error messages
