Panda Gateway

A high-performance, stateless, intent-aware gateway that combines MCP, API, and AI capabilities

Panda is a Rust-based unified gateway that combines MCP (Model Context Protocol) tool orchestration, API gateway capabilities, and AI traffic management. It sits between your clients and services, adding policy, identity, budgets, tool orchestration, caching, safety, and observability in one place.

Key Capabilities

Security & Safety

JWT/JWKS authentication, prompt injection firewall, PII masking, Wasm plugins for custom security policies, and proof of intent for tool calls.

Cost Control

TPM-style token budgets, semantic caching, detailed usage metrics, and optimized streaming to reduce AI spending.

MCP Tools

Model Context Protocol for tool orchestration, tool discovery, execution, aggregation, and seamless integration with external services.

Observability

Prometheus metrics, OpenTelemetry tracing, structured logs, and intent-aware audit logs for comprehensive monitoring.

Extensibility

Wasm plugins for custom policy, transforms, and integrations without forking the core, supporting Rust and Go (via TinyGo).

Performance

Rust-based implementation with async I/O, minimal copying, optimized streaming paths, and horizontal scalability.

API Gateway

Ingress routing, egress client, load balancing, rate limits, TLS management, and portal integration for comprehensive API management.

Intent-Aware

Intent routing, semantic understanding, and context-aware decisions for optimal model selection and tool use.

Deployment Flexibility

Standalone, sidecar, or Kubernetes deployment models with GitOps-friendly configuration and optional Redis for shared state.

Four-Layer Design

  • Ingress Layer - Accepts connections and normalizes protocols
  • Shield Layer - Security and hygiene before downstream work
  • Intelligence Layer - Intent, routing, reuse, and tool surfacing
  • Egress Layer - Adapts to each backend's format
