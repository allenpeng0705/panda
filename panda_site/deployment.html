Deployment

Flexible deployment options for every environment

Deployment Models

Standalone

Single binary with YAML configuration, ideal for small teams and single-region deployments.

When to use

  • • You want one service in front of your LLM upstream
  • • TLS terminates at a cloud load balancer or on Panda
  • • You don't need a large external route/plugin surface

With KONG

Run behind KONG or other API gateways, keeping TLS and coarse routing at the edge.

Integrations

Panda integrates with various tools and services to enhance its capabilities.

Redis

Optional integration for shared TPM counters and semantic cache.

Configuration

# In panda.yaml
tpm:
  redis_url: redis://127.0.0.1:6379

semantic_cache:
  enabled: true
  backend: redis
  redis_url: redis://127.0.0.1:6379

Environment variables:

export PANDA_REDIS_URL='redis://127.0.0.1:6379'
export PANDA_SEMANTIC_CACHE_REDIS_URL='redis://127.0.0.1:6379'

Prometheus

Metrics endpoint for monitoring Panda's performance and usage.

Configuration

Scrape the /metrics endpoint:

# Prometheus config
targets:
  - http://panda:8080/metrics

Protected endpoints:

curl -H "x-panda-admin-secret: $PANDA_OPS_SECRET" http://panda:8080/metrics

OpenTelemetry (OTLP)

Distributed tracing for end-to-end request visibility.

Configuration

export OTEL_EXPORTER_OTLP_ENDPOINT=http://127.0.0.1:4318/v1/traces
export PANDA_OTEL_SERVICE_NAME=panda-gateway
export PANDA_OTEL_TRACE_SAMPLING_RATIO=0.2

Supported backends:

  • • Jaeger
  • • Grafana Tempo
  • • Cloud APM services

Kubernetes Deployment

Panda provides starter manifests for Kubernetes deployments:

Manifests

  • k8s/configmap.yaml - Configuration
  • k8s/deployment.yaml - Deployment
  • k8s/service.yaml - Service
  • k8s/pdb.yaml - Pod Disruption Budget
  • k8s/hpa.yaml - Horizontal Pod Autoscaler
  • k8s/secret.example.yaml - Secret template

Deployment Steps

# Set up configmap
kubectl apply -f k8s/configmap.yaml

# Set up secrets
kubectl apply -f k8s/secret.example.yaml

# Deploy
kubectl apply -f k8s/deployment.yaml
kubectl apply -f k8s/service.yaml
kubectl apply -f k8s/pdb.yaml
kubectl apply -f k8s/hpa.yaml

Configuration

Update the ConfigMap with your specific settings:

# k8s/configmap.yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: panda-config
data:
  panda.yaml: |
    listen: 0.0.0.0:8080
    upstream: http://your-model-server:8000
    identity:
      require_jwt: false
    tpm:
      enforce_budget: true
      budget_tokens_per_minute: 10000
    semantic_cache:
      enabled: true
      backend: memory

Scaling

The Horizontal Pod Autoscaler can scale based on CPU utilization or custom metrics:

# k8s/hpa.yaml
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: panda-hpa
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: panda
  minReplicas: 2
  maxReplicas: 10
  metrics:
  - type: Resource
    resource:
      name: cpu
      target:
        type: Utilization
        averageUtilization: 70

Docker Deployment

Building the Image

# Build the image
docker build -t panda:latest .

# Run the container
docker run --rm -p 8080:8080 \
  -v "$(pwd)/panda.yaml:/app/panda.yaml:ro" \
  panda:latest /app/panda.yaml

Environment Variables

# Example with environment variables
docker run --rm -p 8080:8080 \
  -v "$(pwd)/panda.yaml:/app/panda.yaml:ro" \
  -e PANDA_REDIS_URL=redis://redis:6379 \
  -e OTEL_EXPORTER_OTLP_ENDPOINT=http://otel:4318/v1/traces \
  panda:latest /app/panda.yaml

Docker Compose

For a complete stack with Redis and monitoring:

version: '3.8'
services:
  panda:
    build: .
    ports:
      - "8080:8080"
    volumes:
      - ./panda.yaml:/app/panda.yaml:ro
    environment:
      - PANDA_REDIS_URL=redis://redis:6379
      - OTEL_EXPORTER_OTLP_ENDPOINT=http://otel:4318/v1/traces
    depends_on:
      - redis
      - otel

  redis:
    image: redis:alpine

  otel:
    image: otel/opentelemetry-collector:latest
    volumes:
      - ./otel-config.yaml:/etc/otelcol/config.yaml
    ports:
      - "4318:4318"

TLS Configuration

Option A: TLS at the Load Balancer

Terminate TLS on your cloud load balancer and let Panda listen on HTTP.

  • • Common on AWS, GCP, Azure
  • • Simplifies Panda configuration
  • • Load balancer handles certificate management

Option B: TLS on Panda

Configure Panda to terminate TLS directly:

# In panda.yaml
tls:
  cert_pem: /path/to/cert.pem
  key_pem: /path/to/key.pem

# Or in server block
server:
  address: 0.0.0.0
  port: 443
  tls:
    cert_pem: /path/to/cert.pem
    key_pem: /path/to/key.pem

Security Best Practices

Secrets Management

  • • Use environment variables for secrets
  • • Never commit secrets to git
  • • Use Kubernetes Secrets or a secret manager
  • • Rotate secrets regularly

Access Control

  • • Protect ops endpoints with admin secrets
  • • Enable JWT authentication for API access
  • • Use trusted gateway headers when behind KONG
  • • Restrict network access to Panda

Runtime Security

  • • Run Panda as a non-root user
  • • Set resource limits in containers
  • • Enable Wasm sandboxing for plugins
  • • Keep Panda and dependencies updated
